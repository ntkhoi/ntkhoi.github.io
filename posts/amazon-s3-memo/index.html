MEMO: Amazon S3 - Amazon Simple Storage Service

What is S3.

  • Save place to store your files.
  • Is a object-base storage.
  • The object is spread across multiple devices and facilities.

The basic of S3.

Buckets
  • A bucket is a container of objects stored in Amazon S3.
  • Every object is contained in a Bucket.
  • Bucket name is a univeral namespace. That is, name must be unique globally.
  • Each bucket created in a specific region.
Objects
  • Object are the entities or files stored in S3 bucket.
  • Obejct size is from 0 - 5 TB.
  • A bucket can store unlimit objects.
  • Each objects consists of data(files itself) and metadata (name/value pairs : system metadata and user metadata).
Keys
  • Each object in S3 is identified by a unique identifier call keys (like filename up to 1024 bytes of Unicode)
  • Key must be unique within a single bucket
Object URL

Each object can be address in intenet by a unique URL using web service endpoint, bucket name and object key.

S3 Operations.
  • Create/Deleta a bucket
  • Read/Write a object
  • Delete a object
  • List keys in a bucket

Data in S3 is eventually consistent system. Data will automatically replicated across multiple AZs within a region. Change data may take time to to propagate to all locations.

  • Read after write consistency for PUT a new objects.
  • Eventualy cosistency for overide PUT and DELETE
Access Control

S3 sercure is default (only the owner is able to access S3 when it create ) Can setup access control the the bucket using ACLs

  1. Access Control list (manage access to buckets and object, grant read/write for object and bucket, enabling bucket logging, make bucket host a static web)
  2. Bucket policy (you can specify who can access the bucket, from where (by Classless Inter-Domain Routing [CIDR] block or IP address), and during what time of day.)

Amazon S3 Advanced Features.

Storage Classes
  1. Amazon S3 Standard offers high durability, high availability, low latency, and high performance object storage
  2. Amazon S3 Standard â€“ Infrequent Access (Standard-IA) is the same with standard but design for less frequently access.
  3. Amazon S3 rreduced Redundancy Storage (RRS): offers slightly lower durability (4 nines) than standard or standard I-A at a reduce cost.
  4. Amazon Glacie: Sercure durable , low cost cloud storage for data does not required real time access such as archies and long-term backup.
Object Lifecycle management
  • Automatic moving your objects between dirrerent storage tiers.
  • Can be apply between current version and previous version.
  • Lifecycle configurations are attached to the bucket and can apply to all objects in the bucket or only to objects specified by a prefix.
Encryption
  1. Client-Side Encryption
  2. Server-Side Encrytion
  • SSE-S3 (AWS-Managed Keys)
  • SSE-KMS (AWS-KMS Keys)
  • SSE-C (Customer proviced key)
Versioning
  • Protect by storage multiple version of each object.
  • Once enable, versioning can not be removed from a bucket. Can be suspended.
Pre-Signed URLs
  • the object owner can optionally share objects with others by creating a pre-signed URL.
  • owner can provide your security credentials and specify a bucket name and expiration date and time for URL.
Multipart Upload

Should use multipart upload for objects larger than 100 Mbytes Must use for objects larger then 5GB

Cross-Region Replication
  • Asynchronously replicate all new objects in the source bucket in one AWS region to a target bucket in another region
  • To enable cross-region replication, versioning must be turned on for both source and destination buckets, and you must use an IAM policy to give Amazon S3 permission to replicate objects on your behalf.
  • If turned on in an existing bucket, cross-region replication will only replicate new objects. Existing objects will not be replicated and must be copied to the new bucket via a separate command.
Logging

Including info when enable (disable by default)

  • Request account and IP
  • Bucket name
  • Request time
  • Action PUT/GET/DELETE
  • Response status and error code
Event Notifications

Can be send through.

  • AWS SNS
  • AWS SQS
  • Lamda Function.

Amazon Glacier

Store infrequently data and retrieval time is about 3 - 5 hours.

Archives

In Glacier data stored in archives. a Archive can up to 40 TB and can have unlimitee archive

Vaults

Vaults is container for archive 1 AWS account can have 1000 vaults.

Vaults Locks

Lock policy can be apply to Vault. Can Specify controls such as Write Once Read Many (WORM) in a vault lock policy. Once lock, pocicy can not longer to change.

Data Retrieval

Can retrieve up to 5% of your data stored in Amazon Glacier for free each month
